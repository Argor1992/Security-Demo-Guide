Installation der Demo

Mit dieser Demo können Sie erfahren wie es ist in ein fremdes IT-System einzudringen. Hier werden Sie in die andere Perspektive des Social Engineerings erfahren und was er/sie mit Passwörtern und anderen Daten anfangen kann. Durch unterschiedliche Maßnahmen konnte es den Angreifern gelingen Passwörter zu stehlen, um Quellcode für einen Prototypen zu klauen.

Um die Demo laufen zu lassen benötigen Sie mindestens Java 11. Nachdem Sie Java installiert haben können Sie zum nächsten Schritt gehen. Ein Popup wird Ihnen den Link anzeigen über den Sie die Unternehmenswebsite erreichen können.

Java

Installation der Demo

Jetzt benötigen Sie noch das eigentliche Programm. Es handelt sich hierbei um ein Webportal der Firma Rich Solutions, das als Projektmanagementtool verwendet wird. Solche Plattformen sind natürlich ein gutes Ziel für einen Social Engineer, da hier sehr viele Informationen über Angestellte, Kunden und Projekte ablegen. Ziel ist es den Quellcode für ein bestimmtes Projekt downzuloaden. Ein reales Beispiel könnte Azure Devops sein, dieses Tool ist auch im Internet frei zugänglich und man benötigt lediglich die Zugangsdaten, was ein Kinderspiel ist.

Download

Starten der Demo

Wo Sie jetzt die Demo heruntergeladen haben, müssen Sie jetzt die heruntergeladene zip-Datein entpacken. Navigieren Sie dann in den Ordner und starten sie das Programm companyportal-1.1-SNAPSHOT. Der Start des Programms kann im schlimmsten Fall ein paar Sekunden dauern. Das Programm funktioniert wie gewollt, wenn Sie das folgende Fenster auf Ihrem Desktop sehen.

Übliche Fehler

Wenn die Demo bei Ihnen richtig gestartet ist, können Sie diesen Abschnitt überspringen.

Falls die Demo bei Ihnen nicht startet kann das folgende Ursachen haben: Mindestens Java 11 SDK benötigt - mit dem verlinkten SDK funktioniert es definitiv Mit Konsole testen und Path einrichten um mögliche Ausgabe auszuwerten

Ziel des Social Engineers

Ab jetzt geht es mit der Demo richtig los. Was Sie gerade sehen ist die Loginseite eines Tools der Firma Rich Solutions, das für das Projektmanagement genutzt wird. Solche Tools sind regelmäßig frei im Internet zugänglich ohne VPN und man benötigt lediglich die Zugangsdaten.

Ziel des Hackers ist es hier den Quellode für eine Software zu bekommen. Es geht ihm um ein Programm, dass für Trading verantwortlich ist. Der Quellcode ist für einen Hacker aus vielen Gründen interessant. Zum einen kann er an Konkurrenten verkauft werden oder zum anderen kann er den Code auswerten und nach Sicherheitsschwachstellen suchen. Somit kann er dann später das Programm für seine Zwecke nutzen.
